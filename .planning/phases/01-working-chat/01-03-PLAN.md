## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | GCP credentials in CI | mitigate | Workload Identity Federation per D-18: no long-lived JSON keys in GitHub Secrets. `id-token: write` permission grants the runner an OIDC token that GCP exchanges for short-lived credentials. |
| T-03-02 | Information Disclosure | OPENROUTER_API_KEY in CI logs | mitigate | The key is a masked GitHub secret in the E2E job env and is mounted via `--set-secrets` (Secret Manager reference, not a value) in the Cloud Run deploy step. Neither path writes the value to logs. |
| T-03-03 | Spoofing | Cross-origin requests to Cloud Run | mitigate | `CORS_ORIGIN` env var on Cloud Run is set to the Firebase Hosting URL (D-20). Backend `main.ts` already reads `process.env.CORS_ORIGIN` and passes it to `app.enableCors({ origin })` — any other origin is rejected by the browser. |
| T-03-04 | Tampering | Cross-repo WIF abuse | mitigate | The WIF provider attribute condition restricts to `attribute.repository == '<your-gh-org>/<repo>'` (Task 2 step B3). No other repo can mint a usable token. |
| T-03-05 | Information Disclosure | Firebase service-account JSON | accept | Firebase still requires a long-lived JSON key (`FIREBASE_SERVICE_ACCOUNT`) — Firebase OIDC + WIF is not GA at time of writing. Stored as a masked GitHub secret. Risk accepted given Firebase project scope. |
| T-03-06 | Denial of Service | OpenRouter quota burnt by CI E2E | accept | Each CI E2E run makes 1 LLM call (~$0.0001 with GPT-4o mini). For a solo-dev project this is negligible. Cap protected by an OpenRouter dashboard budget cap (user setup). |
| T-03-07 | Tampering | Public Cloud Run endpoint | mitigate | `--allow-unauthenticated` is intentional (the frontend is public). Mitigations: CORS pinning (T-03-03), no auth-token persistence, no PII stored, rate-limiting deferred to ENH-06. |
| T-03-08 | Repudiation | Deploy actions on main | mitigate | GitHub Actions audit log retains who pushed which SHA to `main` triggered which deploy. WIF principal can only be invoked from this repo per the attribute condition. |
| T-03-SC | Tampering | Third-party GitHub Actions supply chain | mitigate | All actions pinned to MAJOR versions (`@v2`, `@v3`, `@v4`, `@v5`, `@v0`) and sourced from well-known orgs: `actions/`, `google-github-actions/`, `docker/`, `FirebaseExtended/`.
