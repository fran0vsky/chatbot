## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser â†’ NestJS API | Untrusted user-supplied `message` string crosses into the backend at `POST /api/agents/chat` |
| NestJS â†’ OpenRouter | Backend forwards user message to a third-party LLM provider; API key is the secret being protected |
| process.env â†’ AgentsService | `OPENROUTER_API_KEY` loaded from environment; must never be logged or surfaced in HTTP responses |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | AgentsService logging | mitigate | `this.logger.log(...)` includes `threadId` and `message` but NEVER the API key; never log `process.env['OPENROUTER_API_KEY']` or the constructed `ChatOpenAI` instance. Reviewer must confirm no `apiKey` token appears in any log statement. |
| T-01-02 | Information Disclosure | Error responses | accept | Per CONTEXT.md D-05, frontend renders a generic error bubble. NestJS default exception handler returns 500 with no stack to client. Accepted: no global exception filter in this plan (deferred to ENH-07). |
| T-01-03 | Tampering | npm install of `@langchain/openai` | mitigate | `@langchain/openai` is a well-known LangChain-org package (>1M weekly downloads, published by `langchain` org on npm). | T-01-05 | Repudiation | Per-thread memory | accept | `MemorySaver` is in-process and ephemeral. No audit log requirement in Phase 1 (no auth, no users). |
