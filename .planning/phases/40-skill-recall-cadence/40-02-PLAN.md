<design_decisions>
- **State (D-01):** New `readonly activeSkill = signal<{ id: string; title: string } | null>(null);` on the
  chat component. Set in the `skill_active` case; reset to `null` whenever a new conversation starts /
  session changes / chat is cleared (same lifecycle points that reset streaming signals).
- **Render (D-02):** A small hint (one short line / chip), e.g. "Active skill this chat: <title>", shown
  only when `activeSkill()` is non-null. Placed in the Knowledge "Skills" section and/or the "what it
  remembers" surface. Muted, unobtrusive; Tailwind classes consistent with surrounding text.
- **Read-only (D-03):** The hint does not link to edit/delete and does not alter `learnedSkills` or the
  skill-manager component — it is purely informational.
- **Out of scope:** Showing *why* a skill was selected, history of past selections, or any control to
  override the selection. One small hint only.
</design_decisions>
