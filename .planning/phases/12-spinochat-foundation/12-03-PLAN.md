# Plan 12-03: Spinosaurus Mascot Integration

## Objective

Replace the capybara SVG currently inlined in `MessageBubble` (assistant role, lines 62-80
of libs/ui/src/lib/message-bubble/message-bubble.html) with a Spinosaurus-inspired
mascot, and place a larger hero rendering of the same mascot in the landing/empty state
above the input composer. Extract the SVG into a standalone `Mascot` component so the
small (assistant bubble) and large (hero) renderings share one source. Out of scope:
animations (Phase 14, MASC-03/04), background atmosphere (Phase 13).

    with the following contract:

    - selector: `app-mascot`
    - standalone: true
    - changeDetection: ChangeDetectionStrategy.OnPush
    - templateUrl: './mascot.html'
    - One input: `size` of type `'sm' | 'hero'` with default `'sm'`. Use Angular signal-input

    Create libs/ui/src/lib/mascot/mascot.html. It renders an inline SVG with:
    - `role="img"`
    - `aria-label="SpinoChat mascot"`
    - `viewBox="0 0 64 64"`
    - `fill="none"` on the root svg, `fill="currentColor"` on body shapes (so the mascot
      inherits the studio-accent / studio-night-accent text color from its parent — same
      pattern the capybara used)
    - Tailwind size classes bound via Angular class binding:
        - when size === 'sm':  `w-14 h-14` (matches the current capybara dimensions in message-bubble.html line 63)
        - when size === 'hero': `w-32 h-32 md:w-40 md:h-40` (responsive hero)
      Use a `[ngClass]` or `[class.*]` binding off the `size()` signal — do not branch
      with two duplicate `<svg>` elements.

    Spinosaurus silhouette geometry (viewBox 0 0 64 64, all coordinates inside the box):
      - Body (bipedal stance, low to the ground): ellipse cx=32 cy=44 rx=20 ry=9, fill=currentColor
      - Long tapered tail going down-right: polygon points "48,42 62,52 58,54 46,46", fill=currentColor
      - Iconic sail along the back: path "M16,38 Q22,18 32,16 Q42,18 48,38 L46,38 Q40,24 32,22 Q24,24 18,38 Z", fill=currentColor, fill-opacity=0.85
      - Neck rising forward-left: ellipse cx=18 cy=32 rx=6 ry=8, fill=currentColor
      - Long crocodilian snout: polygon points "4,30 18,28 18,36 6,34", fill=currentColor
      - Eye: circle cx=14 cy=30 r=1.4, fill="#1A1D21"
      - Eye highlight: circle cx=14.4 cy=29.6 r=0.5, fill="#FFFFFF"
      - Front leg stub: ellipse cx=24 cy=52 rx=3 ry=4, fill=currentColor
      - Back leg: ellipse cx=38 cy=52 rx=4 ry=5, fill=currentColor
      - Ground shadow: ellipse cx=32 cy=58 rx=18 ry=2, fill="#1A1D21", fill-opacity=0.08

    These coordinates produce a recognizable Spinosaurus silhouette (sail back, long
    snout, bipedal, low tail) within the same 64x64 box as the old capybara — so
    swapping it inside the message bubble does not change layout.
  </action>
