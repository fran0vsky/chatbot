## Spacing Scale

Declared values (multiples of 4, consistent with existing project usage of `p-1`, `py-1`, `pl-3`, `gap-2`, `px-5 py-4`):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px (`p-1`, `gap-1`) | Toggle button hit-padding, inline icon gap inside toggle row |
| sm | 8px (`gap-2`, `mb-2`) | Gap between reasoning block and the markdown content slot below it |
| md | 12px (`pl-3`) | Left padding inside the bordered reasoning region (separates text from the left border) |

Exceptions: none — all values map to the established Tailwind scale already in use across `message-bubble.html`.

## Typography

Only two roles are introduced by this phase; all other text inside the assistant bubble keeps the existing `MessageBubble` typography (`text-sm font-body leading-relaxed`).

| Role | Size | Weight | Line Height | Tailwind |
|------|------|--------|-------------|----------|
| Reasoning body (the trace) | 14px | 400 (regular) | 1.5 (relaxed) | `text-sm font-body leading-relaxed` |
| Toggle label ("Thought for Xs · Show/Hide reasoning") | 12px | 500 (medium) | 1.25 (snug) | `text-xs font-body font-medium leading-snug` |

Whitespace handling for reasoning body: `whitespace-pre-wrap` (plain text — no markdown rendering per SPEC Req 6 / out-of-scope rule).

No italic, no monospace, no display-size text. Total: 2 sizes (12, 14), 2 weights (400, 500). Within the 3-4 sizes / 2 weights gate.

## Color

The phase introduces no new colors. Every value below is an existing token from `apps/frontend/tailwind.config.js`. The 60/30/10 split is inherited from the host bubble; this phase only consumes the **muted secondary** layer.

| Role | Token (day) | Token (night) | Usage in this phase |
|------|-------------|---------------|---------------------|
| Dominant surface (60%) | `bg-desert-parchment` (#EDD9A3) | `dark:bg-desert-night-parchment` (#2E2318) | Inherited from host assistant bubble — NOT redrawn by the reasoning block |
| Secondary / muted text (30%) | `text-desert-brown-muted` (#7A5C42) | `dark:text-desert-night-muted` (#A08C6E) | Reasoning body text AND toggle label — the entire `ReasoningBlock` reads as visually subordinate |
| Border demarcation | `border-desert-border` (#D4B896) | `dark:border-desert-night-border` (#3D2E1C) | Left border (`border-l-2`) that visually quotes the reasoning region from the answer |
| Accent (10%) | `bg-desert-terracotta` / `dark:bg-desert-night-amber` | — | **Reserved for** user-message bubbles, the primary send button, and the active model selector chip. **NOT used inside `ReasoningBlock`** under any condition |
| Destructive | `bg-red-50 / text-red-700 / border-red-200` | (unchanged) | Reserved for error-role messages only (existing). Not used in this phase |
| Focus ring | `focus-visible:ring-desert-gold` (#D4AF37) | `dark:focus-visible:ring-desert-night-border` | Toggle button focus ring — reuses the existing focus-ring tokens applied to the copy/regenerate buttons in `message-bubble.html` |

**Accent reserved for:** user message bubble background, send/submit primary CTA, currently-selected model chip in the model selector. **Never** applied to reasoning text, the reasoning toggle, or the reasoning border. This keeps the reasoning block unambiguously secondary on first glance.

**Background fill rule:** The reasoning block adds NO background tint. Demarcation is via left border + muted text only (per CONTEXT.md D-03). This preserves the host bubble's parchment background and prevents a stacked-layers look.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Toggle label (collapsed, has duration) | `Thought for {N}s · Show reasoning` |
| Toggle label (expanded, has duration) | `Thought for {N}s · Hide reasoning` |
| Toggle label (collapsed, sub-second) | `Thought for <1s · Show reasoning` |
| Toggle label (expanded, sub-second) | `Thought for <1s · Hide reasoning` |
| Toggle label (streaming, no duration yet) | `Thinking… · Hide reasoning` |
| Toggle `aria-label` (collapsed) | `Show model reasoning` |
| Toggle `aria-label` (expanded) | `Hide model reasoning` |
| Toggle `aria-expanded` | bound to `!collapsed` |
| Empty state | none — when `message.reasoning` is empty/absent the entire block is not rendered (no placeholder, no zero-height container, no extra spacing) |
| Error state | none — reasoning-stream errors flow through the existing `error` role message pipeline (`message-bubble.html` error branch). The `ReasoningBlock` itself has no error UI |
| Destructive confirmation | none — this phase has no destructive actions |

**Primary CTA for this phase:** the toggle button is the only interactive element introduced. It is **not** a primary CTA — it is a secondary disclosure control styled at the muted-text contrast level.

**Duration formatting rules (single source — implemented once on `ReasoningBlock.durationLabel()`):**
- `durationMs == null` AND `streaming === true` → render `Thinking…` (no duration segment)
- `durationMs < 1000` → render `<1s`
- otherwise → render `${Math.round(durationMs / 1000)}s`

**Separator:** middle-dot (` · `, U+00B7 with single spaces on both sides). Matches the SPEC Req 8 wording exactly.

## Interaction Contract

| State | Trigger | Behavior |
|-------|---------|----------|
| Initial (streaming starts, reasoning model) | First `reasoning_token` event | Block renders expanded; toggle reads `Thinking… · Hide reasoning`; text streams in token-by-token |
| Auto-collapse | First content `token` event after `streamingReasoning().length > 0` (detection lives in `chat.ts.handleStreamEvent`, per RESEARCH §Pattern map) | `autoCollapsed` input flips to `true`; block transitions to collapsed within 1 frame; toggle updates to `Thought for Xs · Show reasoning` once `done` arrives with `reasoningDurationMs` |
| User toggle | Click on toggle button | Local `userToggle` signal inverts the current `collapsed` computed; once user has clicked, their override takes precedence over `autoCollapsed` for the remainder of the message lifetime |
| Persisted (sidebar revisit) | Component mounted with `streaming=false`, `reasoning` non-empty, `autoCollapsed=true` | Block renders collapsed; user can click toggle to reveal |
| Absent | `reasoning` is empty string or undefined | Block does not render — no DOM nodes, no spacing |

**Animation:** No collapse/expand height animation in this phase (keeps scope minimal and avoids layout thrash during streaming). Toggle simply swaps the body element in/out. If a later phase adds motion, it would use the existing `msg-enter-left` / `msg-enter-right` keyframe approach already in `message-bubble.html`.

**Streaming render cadence:** The reasoning body is signal-bound (`streamingReasoning()` on the host) so OnPush change detection fires per token append — same cadence as Phase 10's `streamingText`. No throttling, no batching.

## Component Shape

`ReasoningBlock` (presentational, standalone, OnPush) — single new component introduced by this phase.

**Selector:** `app-reasoning-block`
**Location:** `libs/ui/src/lib/reasoning-block/`
**Inputs (no outputs — toggle state is local):**

| Input | Type | Required | Notes |
|-------|------|----------|-------|
| `reasoning` | `string` | yes | The full (or in-progress) reasoning text |
| `streaming` | `boolean` | no (default `false`) | When `true`, suppresses `autoCollapsed` and renders `Thinking…` label until duration arrives |
| `durationMs` | `number \| undefined` | no | Drives the duration segment of the toggle label |
| `autoCollapsed` | `boolean` | no (default `false`) | Parent-driven collapse signal; overridden by user click |

**Host integration:** Slotted inside both the persisted assistant bubble (in `message-bubble.html`, above `<markdown>`) AND the in-progress streaming assistant bubble (location confirmed by planner per RESEARCH Open Question 1). Gated on `message.reasoning` (persisted) or `streamingReasoning().length > 0` (streaming).

**Storybook story (Req 5 acceptance, project profile requirement for presentational components):**
- Story 1: `collapsed` — short reasoning, duration 4s, autoCollapsed=true, streaming=false
- Story 2: `expanded` — long multi-paragraph reasoning, duration 12s, autoCollapsed=false, streaming=false
- Story 3: `streaming` — partial reasoning text, durationMs undefined, streaming=true (label reads `Thinking… · Hide reasoning`)
- Story 4: `sub-second` — short reasoning, durationMs 450, autoCollapsed=true, streaming=false (label reads `Thought for <1s · Show reasoning`)
- Story 5: `night-mode` — Story 1 rendered with `.night-mode` class on the decorator wrapper (covers Req 6 dark-mode verification)
