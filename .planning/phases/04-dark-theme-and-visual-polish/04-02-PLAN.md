    Add theme toggle state and logic to ChatComponent. All changes are additive — existing members
    are untouched except ngOnInit gains new lines.

    New class member to add (after `placeholder` property declaration):

      isDayMode = true;

    New private method `applyTheme(mode: 'day' | 'night'): void`:
      - Sets this.isDayMode = (mode === 'day')
      - Calls document.documentElement.classList.remove('day-mode', 'night-mode')
      - Calls document.documentElement.classList.add(mode === 'day' ? 'day-mode' : 'night-mode')
      - Calls this.cdr.markForCheck()

    New public method `toggleTheme(): void`:
      - Reads const next = this.isDayMode ? 'night' : 'day'
      - Calls localStorage.setItem('desert-theme', next)
      - Calls this.applyTheme(next)

    Modify ngOnInit(): add these lines AT THE START of the method body (before the placeholder timer).
    IMPORTANT: only sync the isDayMode property here — do NOT call applyTheme(). The theme class is
    already set on &lt;html&gt; by the inline script in index.html before Angular bootstraps. Calling
    applyTheme() in ngOnInit would be redundant and could trigger an unnecessary class toggle:

      const saved = localStorage.getItem('desert-theme') as 'day' | 'night' | null;
      this.isDayMode = saved !== 'night';

    This single assignment syncs the isDayMode boolean (so the sun/moon button icon is correct) without
    touching document.documentElement.classList. The classList was already set correctly by the inline
    script. applyTheme() continues to be called only from toggleTheme() at runtime.

    No changes needed to ngOnDestroy, send(), newChat(), or any other existing method.

    TypeScript type safety: localStorage.getItem returns string | null. Cast to 'day' | 'night' | null
    as shown. The expression `saved !== 'night'` safely defaults to true (day mode) for null or any
    unexpected value — do not use `as` cast on the boolean result.

    No new imports required (document is a global in browser context; no need to import DOCUMENT token
    for this simple usage pattern).
  </action>
