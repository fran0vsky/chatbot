    Replace the current file content entirely. The new file must:

    1. Keep the three @tailwind directives (base, components, utilities) at the top.

    2. Load both Prism theme CSS files unconditionally at the root level (NOT nested inside a selector).
       Nesting @import inside a rule block (.day-mode { @import ... }) is non-standard Dart Sass and
       will fail with "CSS @import can only be used at the root of a stylesheet". Use root-level imports:

         @import 'prismjs/themes/prism-solarizedlight.css';
         @import 'prismjs/themes/prism-tomorrow.css';

       Both themes are loaded, but the night theme wins in night mode because we add scoped overrides
       that give .night-mode pre/code elements higher specificity. Use these selectors immediately after
       the @import lines to ensure night Prism styles win when .night-mode is active:

         /* Night mode: ensure prism-tomorrow theme wins over prism-solarizedlight */
         .night-mode pre[class*="language-"],
         .night-mode code[class*="language-"] {
           background: #2d2d2d;
           color: #ccc;
         }

       Day mode (.day-mode) wins by default because prism-solarizedlight is imported first and sets
       the base styles; prism-tomorrow's rules are then overridden back to solarizedlight values under
       .day-mode if needed, but in practice loading order + night-mode specificity override is sufficient.

    3. Remove the original unconditional `@import 'prismjs/themes/prism.css'` line (not present after
       step 2 replaces it, but verify it does not reappear).
