<objective>
Add the avatar upload half of the Custom Dino Creator: a multipart `POST /api/custom-dinos/avatar`
endpoint that streams an image to a public-read GCS bucket and returns the public URL the creation
form stores in `avatarUrl`. Provision the bucket + IAM. This is the only deliverable here — no
resolution wiring (Plan 03) and no UI (Plan 04). Multipart (not base64-in-JSON) is deliberate so the
upload never depends on the Phase 38 body limit and never bloats chat payloads (42-CONTEXT, LOCKED).

<design_decisions>
- **Dependency (D-01):** Add `@google-cloud/storage` (runtime) and `@types/multer` (dev) to the root `package.json`. The Storage client picks up Application Default Credentials automatically on the VM (`--scopes=cloud-platform`), so no credentials/env beyond `AVATAR_BUCKET`.
- **Endpoint (D-02):** `POST /custom-dinos/avatar`, `@UseInterceptors(FileInterceptor('file'))`, `@UploadedFile() file`. Returns `{ url: string }`. Lives in a dedicated `AvatarController`/`AvatarService` pair (one responsibility per class), registered in `AgentsModule`.
- **Validation (D-03):** Reject when no file, when `file.mimetype` does not start with `image/`, or when `file.size > 2 * 1024 * 1024` — all `BadRequestException` with specific messages. Validate BEFORE touching GCS.
- **Object naming (D-04):** Upload to `avatars/<uuid>.<ext>` (ext derived from mimetype: png/jpeg/jpg/gif/webp). Set the object `contentType` to the file mimetype. Public URL is `https://storage.googleapis.com/<AVATAR_BUCKET>/avatars/<uuid>.<ext>` (bucket is public-read via uniform access, so no per-object ACL call is needed).
- **Graceful degradation (D-05):** If `process.env['AVATAR_BUCKET']` is unset/empty, the service throws `BadRequestException('avatar upload is not configured')` — the creation form surfaces this and blocks avatar selection, but the backend still builds and boots without GCS (local/dev/e2e). The Storage client is constructed lazily inside `upload()` (not at module load) so an unconfigured env never breaks boot.
- **Infra (D-06):** In `provision-gcp.sh` add an `AVATAR_BUCKET`-named bucket to the existing public-read bucket loop (or a parallel block), and add `roles/storage.objectAdmin` to the VM SA role loop. The VM today has only `artifactregistry.reader` + `secretmanager.secretAccessor`. Record the bucket + IAM + `AVATAR_BUCKET` contract in `.planning/INFRASTRUCTURE.md`.
- **Scope fence:** This plan does NOT modify `custom-dinos.service.ts`/`controller.ts` (avatarUrl is already an accepted string field), the chat loop, or any frontend file.
</design_decisions>
