<objective>
Extend the backend to route each chat request to the LLM model selected by the user.

Three concrete changes:
1. Add `model?: string` to the shared `ChatRequest` interface (backwards-compatible optional field).
2. Refactor `AgentsService` from a single `ChatOpenAI` instance to a `Map<string, CompiledStateGraph>` — one compiled graph per supported model ID, all sharing the same `MemorySaver` so thread history is preserved across model switches.
3. Update `AgentsController.chat()` to pass `body.model` through to `runAgent()`.

After this plan, the backend honours the model field from every request. The frontend wiring (Plan 02-02) is a separate wave.
</objective>

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → NestJS API | User-supplied `model` string now arrives in `ChatRequest.model`; must not be used to construct arbitrary HTTP requests to OpenRouter |
| NestJS → OpenRouter | Backend forwards `model` as the OpenRouter model ID; fallback gate prevents unknown IDs from reaching the API |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-02-01 | Elevation of Privilege | AgentsService model selection | mitigate | Unknown model IDs are silently fallen back to `openai/gpt-4o-mini` (D-11) — the `graphs.get(model) ?? graphs.get('openai/gpt-4o-mini')!` fallback ensures only pre-built graphs (for the two supported models) are ever used. No arbitrary model ID is passed to OpenRouter. |
| T-02-02 | Information Disclosure | Logger in runAgent | mitigate | Logger may log `message` and `threadId`; verify it does NOT log `body.model` in a way that exposes user intent beyond operational debugging. Model ID is not sensitive but should not be logged at DEBUG level unnecessarily. |
| T-02-03 | Tampering | ChatRequest backwards compatibility | accept | Adding `model?: string` to `ChatRequest` is backwards-compatible. Existing Phase 1 E2E test sends no `model` field; the fallback in `runAgent` ensures it still works. |

</threat_model>
