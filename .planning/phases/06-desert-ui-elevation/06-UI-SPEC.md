# Phase 6 — UI Design Contract

> Visual and interaction contract for Phase 6: Desert UI Elevation.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase is purely aesthetic — no new functionality. Four work areas:
> 1. Pixel snake avatar: 24×24 profile/S-curve SVG replacing current 8×8 art
> 2. Decorative background cactus silhouettes in the chat area
> 3. Floating pill-style chat input (rounded-2xl, shadow, centered, ~55% width)
> 4. Typography upgrade: Playfair Display → Cinzel, all-caps CHATBOT

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, snake avatar margin (`mb-1`) |
| sm | 8px | Button padding, pill inner vertical padding (`py-2`) |
| md | 16px | Pill horizontal padding (`px-4`), message area padding (`px-4`), pill bottom gap (`pb-4`) |
| lg | 24px | Section padding (`py-6` on wrapper if needed) |
| xl | 32px | Send button touch target (inherits `w-8 h-8` = 32px from Phase 5) |
| 2xl | 48px | Not used in Phase 6 additions |
| 3xl | 64px | Not used in Phase 6 additions |

Exceptions:
- Pill width: `w-[55%]` desktop / `w-[90%]` mobile — intentionally narrow to reserve sidebar space (D-11)
- Pill bottom padding: `pb-4` or `pb-5` on sticky wrapper (16–20px gap below pill) (D-13)
- Send button: `w-8 h-8` (32px) circular — smaller than current `w-10 h-10` square (D-15)
- Snake avatar rendered at 24×24 px (`width="24" height="24"`) (D-04)
- Cactus SVG positioning: `absolute` with per-silhouette offsets (bottom-left, bottom-right, etc.) — no standard token, use Tailwind positional utilities (`bottom-0 left-0`, `bottom-0 right-0`, `top-8 right-4`, etc.)

### New UI Elements

#### 1. Snake Avatar SVG (24×24)

Replaces the current 8×8 viewBox snake in `message-bubble.html`.

Specification:
- `width="24" height="24"` on the `<svg>` element
- `viewBox="0 0 24 24"`
- `shape-rendering="crispEdges"` — mandatory for pixel art crispness (D-04)
- `aria-hidden="true" focusable="false"` — decorative, screen readers skip it
- Orientation: **profile/S-curve** — head facing right, S-curve body, tail tapering left/down (D-01)
- Head: visibly wider pixel block (2–3px wide), distinct from body segments
- Eye: single dark pixel (`#1A1209`) on the head (D-03)
- Tongue: 2-pixel forked extension from snout tip, `#C1644A` terracotta fill (D-02)
- Body fill: `#4A7C59` (`cactus-green`) for main body segments (D-03)
- Highlight pixels: `#6BAF82` (`cactus-green-light`) on upper body segments for shading (D-03)
- Tail: tapers — single pixel wide at tip
- Implementation: `<rect>` elements per pixel, same pattern as existing snake

Pixel layout guide (24×24 grid, head right, S-curve):
```
Row  0: ........................
Row  1: ......................HH   ← head (cols 22-23), 2px wide
Row  2: ...............BBBBBBHE   ← body + head + eye pixel
Row  3: .........BBBBBBB.......   ← S upper curve
Row  4: .......BB..............   ← S mid
Row  5: .......BBBBBBB.........   ← S lower curve
Row  6: .............BBBBBBB...   ← tail segment
Row  7: ...................BBB..
Row  8: ......................B.   ← tail tip
```
(H=head fill, E=eye pixel, B=body fill — exact pixel coords are executor discretion within this layout intent)

#### 2. Cactus Silhouettes (4–5 SVGs in `<main>`)

Structure per silhouette:
```html
<svg
  class="absolute pointer-events-none select-none opacity-[0.05] dark:opacity-[0.03] text-desert-brown dark:text-desert-night-text"
  aria-hidden="true"
  focusable="false"
  ...positional classes...
>
  <path d="..." fill="currentColor" />
</svg>
```

Positioning layout (5 silhouettes):
```
┌─────────────────────────────────────────┐
│                              [small]    │  ← top-right: small barrel/prickly-pear
│                                         │
│  [small]                                │  ← mid-left: small oval barrel
│                                         │
│                                         │
│                         [medium]        │  ← mid-right: medium branched cactus
│                                         │
│ [tall saguaro]       [prickly pear]     │  ← bottom-left, bottom-center-right
└─────────────────────────────────────────┘
```

Tailwind positional classes per silhouette:
| Silhouette | Position classes | Size | Shape |
|------------|-----------------|------|-------|
| Tall saguaro | `bottom-0 left-4` | `w-24 h-48` | Classic two-arm saguaro |
| Prickly pear | `bottom-0 right-16` | `w-20 h-24` | Two oval pads stacked |
| Mid-right branched | `top-1/3 right-8` | `w-16 h-32` | Single-arm branched |
| Top-right small | `top-8 right-4` | `w-10 h-16` | Thin barrel |
| Mid-left small | `top-1/2 left-8` | `w-12 h-20` | Squat barrel oval |

- `z-0` on each SVG; message content sits at default stacking order (above z-0)
- `<main>` must have `relative` class added (D-06; code context note)
- All paths are simple filled silhouettes — no stroke, no detail lines (D-specifics)

## Constraints Summary

All Phase 6 UI work must comply with:

1. **Tailwind-only** — no inline styles, no `style="..."` attributes. All new classes must be Tailwind utilities or existing desert/cactus palette tokens.
2. **Angular standalone + OnPush** — Phase 6 adds no new state fields. No `cdr.markForCheck()` calls needed (no new bindings or flag toggles).
3. **No new npm packages** — Google Fonts CDN only. Zero npm dependency additions.
4. **Inline SVG** — snake avatar and all cactus silhouettes are inline `<svg>` in HTML templates. No external SVG files, no `<img src="...">`, no icon fonts.
5. **Desert palette tokens only** — no arbitrary hex values in Tailwind class attributes. Snake pixel fills use hardcoded hex inside `<rect fill="...">` attributes only (pixel art SVG exception — same pattern as existing snake); all Tailwind classes use named tokens.
6. **`darkMode: ['class', '.night-mode']`** — all dark variants use `dark:` prefix which resolves to `.night-mode` class. No `prefers-color-scheme` media queries.
7. **TypeScript strict mode** — `noImplicitAny`, `strictTemplates`. Phase 6 changes are HTML/SVG/config only — no new TypeScript logic required.
8. **Messages stay full-width** — pill narrowing (`w-[55%]`) applies to the input container only. Message bubbles keep `max-w-[75%]` left/right alignment unchanged (D-14).
9. **No functional regressions** — chat send, model selector, theme toggle, new chat, copy buttons, code block headers all unchanged.

Source: `apps/frontend/CLAUDE.md`; `.planning/codebase/CONVENTIONS.md`; 06-CONTEXT.md constraints section
