<objective>
Eliminate the stale-message flash when switching conversations. Today `switchToSession()` swaps the
message list via the synchronous `switchSession` reducer; cover that transition with a transient
skeleton so the message area never shows the outgoing thread's bubbles.

Purpose: the app feels reliable — chat switches read as a clean transition, not a flicker of the wrong chat.
Output: a `threadSwitching` flag on ChatComponent and a skeleton branch in the message-list region.
</objective>

<design_decisions>
- A boolean field `threadSwitching` (or signal) on ChatComponent gates the skeleton. Set true at the top of
  `switchToSession`, before the dispatch; clear it after the new messages are committed using `requestAnimationFrame`
  (or `queueMicrotask` + `cdr.markForCheck()`), so the skeleton occupies the transition frame and the prior
  thread's bubbles are never painted.
- Skeleton is Tailwind `animate-pulse` placeholder rows (a few rounded bars) — match existing surface colors
  (`jungle-surface`/`jungle-night-surface`). No inline styles (project rule). Reuse a `libs/ui` skeleton primitive
  if one already exists; otherwise inline the markup in chat.html.
- Only the scrolling message region is covered; the dino header and composer stay mounted.
</design_decisions>
