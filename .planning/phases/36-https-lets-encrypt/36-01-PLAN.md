<design_decisions>
- **Host-level nginx, HTTP-bootstrap config (D-01, D-02):** Ship `infra/nginx/dinoagents.conf` as a
  single `server { listen 80; server_name {DOMAIN}; ... }` block that already reverse-proxies to
  `localhost:3000`. This lets nginx start and serve over HTTP before any cert exists. The runbook
  then runs `sudo certbot --nginx -d {DOMAIN}`, which obtains the cert and **automatically rewrites
  the file** to add a `listen 443 ssl` server, the cert paths under
  `/etc/letsencrypt/live/{DOMAIN}/`, and a `301` redirect on the port-80 block — plus the systemd
  renew timer. Do NOT hand-author a 443 block referencing a non-existent cert (nginx won't start).
- **Streaming + upload safety (D-04):** In the proxy location, set
  `proxy_http_version 1.1`, `proxy_set_header Host $host`, `X-Real-IP`, `X-Forwarded-For`,
  **`X-Forwarded-Proto $scheme`** (so the app sees https), `Upgrade $http_upgrade` +
  `Connection "upgrade"`, `proxy_buffering off`, `proxy_cache off`, and
  `proxy_read_timeout 3600s` / `proxy_send_timeout 3600s` for long SSE streams. Set
  `client_max_body_size 25m` (server-level) for pasted screenshots. The backend's
  `X-Accel-Buffering: no` already disables buffering per-response; `proxy_buffering off` is
  belt-and-suspenders.
- **CORS (D-05):** `docker-compose.yml` backend `environment` gains
  `- CORS_ORIGIN=${CORS_ORIGIN:-https://{DOMAIN}}` and `.env.example` documents
  `CORS_ORIGIN=https://your-domain.example`. The maker sets the real value in the VM's `.env`.
- **README rewrite (D-06):** Replace the whole `## Deployment` section. New content: architecture
  (VM runs the backend container behind host nginx+certbot, SPA served same-origin), prerequisites
  (domain A record → VM IP), and a numbered runbook. Keep it accurate to the repo (single
  container, port 3000, `CORS_ORIGIN`). Drop Cloud Run, Artifact Registry, WIF, Firebase Hosting,
  and the GitHub-Actions GCP variable tables that describe the retired architecture.
- **Placeholder convention (D-03):** Use `{DOMAIN}` (or `your-domain.example`) everywhere; commit
  no real hostname or cert.
</design_decisions>

<threat_model>
| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-36-01-01 | Information Disclosure | Plaintext HTTP traffic | mitigate | TLS termination at nginx; port-80 server block redirects (301) to HTTPS so no app traffic is served in plaintext after issuance. |
| T-36-01-02 | Spoofing | Certificate validity | mitigate | Let's Encrypt domain-validated cert via certbot HTTP-01; `certbot renew --dry-run` + systemd timer ensure it never silently expires. |
| T-36-01-03 | Tampering | X-Forwarded-* headers | accept | nginx sets `X-Forwarded-Proto`/`X-Real-IP` from the trusted local proxy; backend binds `localhost:3000` only (not public), so headers can't be spoofed from outside the proxy. Document that 3000 must not be exposed in the VM firewall. |
| T-36-01-04 | Denial of Service | Large upload bodies | mitigate | `client_max_body_size 25m` caps request size at the edge before it reaches the app; oversized uploads are rejected by nginx, not buffered by the backend. |
| T-36-01-05 | Information Disclosure | CORS misconfig | mitigate | `CORS_ORIGIN` pinned to the exact `https://{DOMAIN}` origin (not `*`, not the localhost dev default) in production. |
</threat_model>
