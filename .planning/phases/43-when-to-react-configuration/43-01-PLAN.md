<design_decisions>
- **Contract (D-01):** `ReactionLevel = 'never' | 'rarely' | 'normal' | 'chatty'`; `REACTION_LEVELS` is the ordered readonly array (single source of truth for validation + UI). `DinoReactivityMap = Record<string, ReactionLevel>`. Put these in `dino.types.ts` (per-dino config).
- **Table (D-08):** `dinoReactivity` columns: `userId text notNull`, `dinoId text notNull`, `level text notNull`, `createdAt`, `updatedAt`. A **unique** index on `(userId, dinoId)` (enables upsert / one row per dino) plus the existing per-table convention. No column on `customDinos` — `custom:`-prefixed ids store rows here exactly like built-in ids.
- **Service (D-08):** `ReactivityService` mirrors `MemoryService`: inject `DATABASE_CONNECTION`, null-db/missing-userId guards, try/catch that logs + returns `{}` (get) / no-ops (set) on infra error. `getLevels(userId)` returns a `DinoReactivityMap`. `setLevel(userId, dinoId, level)` validates `level ∈ REACTION_LEVELS` (else `BadRequestException`) then upserts on the unique `(userId, dinoId)` index.
- **Endpoints (D-09):** thin `ReactivityController`: `GET /dino-reactivity?userId=` → `{ levels }`; `PUT /dino-reactivity/:dinoId` body `{ userId, level }` → `{ dinoId, level }`.
- **Nudge (D-01/D-05/D-06):** `buildDecisionPrompt(profile, threadText, hasPrior, level: ReactionLevel = 'normal')`. `normal` adds **no** line (output identical to today → existing decision.spec stays green). `rarely` adds a line like "You tend to stay quiet — only speak when you clearly add something; otherwise prefer silent/react." `chatty` adds "You love to chime in — lean toward answering or at least reacting." `never` never reaches here (short-circuited). Persona lines still come first → the level governs FREQUENCY while the persona governs CONTENT (D-06 precedence, SC#3); a code comment at the nudge-append site states this.
- **Clamp (D-02/D-03):** In `streamGroup`'s decide block, resolve `level = levels.get(dino.id) ?? 'normal'`. If `level === 'never' && !isForcedAnswer`, set `decision = { action: 'silent' }` BEFORE the image-gen / LLM branches — a deterministic hard stop that also silences the image-gen react path and saves the LLM decision call. Because the clamp sits at the decision result site, it also covers the `heuristicDecision` fallback path (degraded paths stay consistent, D-03). The hook is confined to `buildDecisionPrompt` + this clamp — **no change to the round/cost-cap machinery or the SSE event contract** (D-03). `@mentioned` dinos (`isForcedAnswer`) remain a documented exception (an explicit per-message address beats standing config, D-06's "never is absolute" applies only to autonomous decisions), like the image-gen exception.
- **Resolution (D-04):** `streamGroup` already has `userId`; resolve the levels map once after `resolveRoster`, before the round loop, and read from it each iteration. Default `{}` when null-db so behavior is unchanged locally/e2e.
- **No auto-migration (project rule):** This repo has no migration runner — Cloud SQL schema is applied manually and local/e2e run null-db (memory: "No DB Migrations"). Do NOT add a drizzle-kit push task; the data path degrades to defaults without the table. The manual Cloud SQL column/table add is recorded as a non-blocking deploy note in `<verification>`.
</design_decisions>
