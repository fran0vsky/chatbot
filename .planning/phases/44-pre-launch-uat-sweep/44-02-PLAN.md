<design_decisions>
- **Location:** root-level `KNOWN_LIMITATIONS.md` (no `docs/` directory exists; root keeps it discoverable and linkable from README).
- **Two registers:** a short user-facing list first (what a first user actually experiences), then an "Operational notes" appendix (the internal cause of each, for the operator/dev). Keeps SC#3's "short note for first users" promise while preserving traceability.
- **No fixes here:** this plan only documents reality. Anything a tester upgrades to a blocker is fixed in Plan 04, and the note can be updated then; it is not a substitute for fixing blockers.
- **Plain language:** user-facing entries avoid internal terms (no "TAVILY", no "413") — those live only in the appendix.
</design_decisions>
