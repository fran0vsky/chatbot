executes every Tier-0 and Tier-1 check on https://dinoagents.duckdns.org, records PASS/FAIL +

a person interacting with the production UI; an agent cannot click through prod.

<design_decisions>
- **Human-gated:** `autonomous: false` because the work is interacting with the live app. The executor's role is to drive/record alongside the human and apply the triage routing — not to fabricate PASS results.
- **Tier-0 first, gate the rest:** if any Tier-0 first-touch feature is a blocker, that is the priority signal for Plan 04; Tier-1 still runs to surface the full picture, but Tier-0 blockers rank highest.
- **Triage routing is explicit:** blocker → a single "Blockers for Plan 04" section appended to the runbook (so Plan 04 has a concrete input); minor → STATE Pending Todos (or the relevant phase's deferred-items.md). Known-issues are excluded from new filing.
- **Close swept UAT items:** for each phase whose UAT passes, mark its pending HUMAN-UAT entry in STATE resolved so the backlog actually shrinks.
</design_decisions>
