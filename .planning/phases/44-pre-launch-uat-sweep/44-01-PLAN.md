<design_decisions>
- **Single artifact, in the phase dir:** the runbook lives at `.planning/phases/44-pre-launch-uat-sweep/44-PROD-UAT-RUNBOOK.md`. It is the operator's worksheet — Plan 03 writes results back into it.
- **Retarget, don't reinvent:** transcribe each source script's steps verbatim in intent, but replace any `nx serve` / localhost setup with "on https://dinoagents.duckdns.org" since prod is already deployed. Drop steps that only make sense locally (e.g. `drizzle-kit push`, unset `DATABASE_URL`) and add a one-line note that the DB-unset degradation path is a localhost-only check, not a prod check.
- **MVP framing (Mode: mvp):** the vertical "slices" of this phase are the per-feature UAT tiers. Tier-0 (the five first-touch features) is the thinnest end-to-end proof and runs first; the full backlog is Tier-1.
- **Result schema is fixed and inline:** every check is a markdown row/checklist entry with `Result: [ PASS / FAIL ]`, `Severity: [ blocker / minor ]`, `Defect:` so the sweep needs no separate tracking sheet.
- **Known-issues callout prevents wasted triage:** the three already-documented prod gaps (web_search, 413 body limit, DB drift) are listed up front as "expected / do not re-file as new blockers."
</design_decisions>
