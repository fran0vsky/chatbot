<objective>
Refine the chat input textarea auto-resize. Two targeted fixes (D-04, D-06):

1. Replace the hardcoded `lineHeight = 24` in `autoResize()` with a value derived from the
   textarea's computed style, so the 5-line cap matches the actual rendered line height.
2. When the textarea reaches its max height and content still overflows, switch from the
   always-on `overflow-hidden` to `overflow-y-auto` so the user can scroll inside the textarea.
   Below max height, keep `overflow-hidden`.

This is a targeted fix, not a rewrite — Enter-to-send / Shift+Enter behavior (D-05) is untouched.

Output: updated chat-input.ts and chat-input.html.
</objective>

  <action>
    Add a public boolean field `atMaxHeight = false;` next to `draft`.

    Rewrite `autoResize(textarea: HTMLTextAreaElement): void` to:
      - Set `textarea.style.height = 'auto'` (unchanged — lets scrollHeight shrink).
      - Derive line height from computed style:
          const computed = getComputedStyle(textarea);
          const parsed = parseFloat(computed.lineHeight);
          const lineHeight = Number.isNaN(parsed) ? 24 : parsed;
        `lineHeight` can be the string `'normal'` → `parseFloat` returns NaN; fall back to 24.
      - const maxRows = 5;
      - const maxHeight = lineHeight * maxRows;
      - const target = Math.min(textarea.scrollHeight, maxHeight);
      - textarea.style.height = target + 'px';
      - this.atMaxHeight = textarea.scrollHeight > maxHeight;

    Note: `submit()` already calls `autoResize` after clearing the draft, which will reset
    `atMaxHeight` to false when the textarea is emptied — no extra reset needed.

    OnPush note: `autoResize` is triggered by the `(input)` DOM event, which is an event handler
    bound in the template — Angular runs change detection for the component after a template
    event handler returns, so the `atMaxHeight` class binding updates without an explicit
    `markForCheck()`. Do not inject ChangeDetectorRef for this.

    Keep `submit()` and `onKeydown()` exactly as they are. No `any` types.
  </action>
