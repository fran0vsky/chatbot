# Phase 5 — UI Design Contract

> Visual and interaction contract for Phase 5: Further UI/UX Work.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase adds three targeted UX polish items to the desert-themed chatbot:
> 1. Message entry animations (slide+fade, 150ms)
> 2. Textarea overflow fix at max height (overflow-y-auto at 5-line cap)
> 3. Code block header bar (language label left, copy button right, always visible)

## Animation Contract

Phase 5 introduces one motion token. All other transitions remain as established in Phase 3/4.

| Element | Animation | Duration | Easing | Trigger |
|---------|-----------|----------|--------|---------|
| New user bubble | Slide in from right + fade in | 150ms | `ease-out` | Message appended to list during active session |
| New assistant bubble | Slide in from left + fade in | 150ms | `ease-out` | Message appended to list during active session |
| Typing indicator bubble | Slide in from left + fade in | 150ms | `ease-out` | `typing` input set to true during active session |
| Greeting message (on load) | None — instant | — | — | Already present on initial render |
| Post-new-chat messages | None — instant | — | — | After chat reset, no cascade animation |

**Slide distance:** `translateX(16px)` (right → 0 for user, left → 0 for assistant). Small enough to feel snappy, not cartoonish.

**Implementation discretion (05-CONTEXT.md):** Angular `@trigger` animations vs. CSS `@keyframes` toggled by `[class.is-new]` — executor picks whichever integrates cleanest with `MessageBubble` + OnPush. Recommended: CSS keyframes via `is-new` class flag — avoids Angular animations import overhead, works cleanly with OnPush via `markForCheck()`.

**OnPush constraint:** Any `isNew` flag toggle that drives animation must call `cdr.markForCheck()` immediately after setting. See established pattern in `message-bubble.ts` `copyMessage()`.

Source: 05-CONTEXT.md D-01, D-02, D-03; Specifics section "Animation feel: Snappy 150ms"

### New UI Elements

#### Code Block Header Bar

Structure (rendered inside each `<pre>` wrapper):
```
┌─────────────────────────────────────┐
│ typescript                    [copy] │  ← always visible, 32px height
├─────────────────────────────────────┤
│  const x = 1;                       │  ← existing Prism-highlighted code
└─────────────────────────────────────┘
```

- **Container:** `flex items-center justify-between px-3 py-2` — parchment bg day / night-parchment bg night
- **Bottom border:** `border-b border-desert-border dark:border-desert-night-border` (separator between header and code)
- **Language label:** `text-xs text-desert-brown-muted dark:text-desert-night-muted font-mono` — shows detected language string (e.g., `typescript`, `bash`). If no language detected, label slot is empty (zero-width, copy button still shown right-aligned).
- **Copy button:** `p-1 rounded text-desert-brown-muted dark:text-desert-night-muted hover:bg-desert-border dark:hover:bg-desert-night-border hover:text-desert-brown dark:hover:text-desert-night-text transition-colors` — matches existing message bubble copy button style exactly
- **Copy icon:** same inline SVG copy/checkmark pair as `message-bubble.html` (established Phase 3 pattern)
- **Copy feedback:** icon swaps to checkmark for 2000ms then reverts — matches `copyMessage()` timing in `message-bubble.ts`
- **Copies:** raw code text only (no HTML, no syntax tokens) — `textContent` of the `<code>` element

Source: 05-CONTEXT.md D-07, D-08, D-09, D-10; existing `message-bubble.html` copy button classes

#### Textarea Overflow State

Current: `overflow-hidden` always applied (hardcoded class on textarea).

Phase 5 contract: textarea class switches based on whether max height is reached.

| State | Class | Condition |
|-------|-------|-----------|
| Below max height | `overflow-hidden` | `scrollHeight < maxHeight` |
| At max height | `overflow-y-auto` | `scrollHeight >= maxHeight` |

`maxHeight` = `lineHeight * 5` where `lineHeight` is derived from `getComputedStyle(textarea).lineHeight` (not hardcoded `24`).

Source: 05-CONTEXT.md D-04, D-06

## Interaction States

### Code Block Copy Button

| State | Visual |
|-------|--------|
| Idle | Copy SVG icon, `text-desert-brown-muted` / `dark:text-desert-night-muted` |
| Hover | `hover:bg-desert-border dark:hover:bg-desert-night-border`, `hover:text-desert-brown dark:hover:text-desert-night-text` |
| Active (0–2000ms after click) | Checkmark SVG icon, same color tokens |
| After 2000ms | Reverts to Copy SVG icon |

### Textarea at Max Height

| State | Overflow class | Scrollbar |
|-------|---------------|-----------|
| Lines 1–4 (below max) | `overflow-hidden` | Hidden |
| Line 5+ (at/above max) | `overflow-y-auto` | System default thin scrollbar |

## Constraints Summary

All Phase 5 UI work must comply with:

1. **Tailwind-only** — no inline styles, no `style="..."` attributes. All new classes must be Tailwind utilities or use existing desert palette tokens.
2. **Angular standalone + OnPush** — every new component state change (animation flag, copy state, overflow toggle) must call `cdr.markForCheck()`.
3. **No new dependencies** — ngx-markdown and Prism.js are already installed. No new npm packages unless executor finds no viable path with existing tools (must flag to user).
4. **Inline SVG icons** — copy/checkmark icons follow established `message-bubble.html` pattern exactly. No icon font, no external SVG files.
5. **Desert palette tokens only** — no arbitrary hex values in templates. Use `desert-*` and `cactus-*` Tailwind tokens defined in `tailwind.config.js`.
6. **TypeScript strict mode** — `noImplicitAny`, `strictTemplates` apply. All new code must pass lint without `any`.

Source: `PROJECT.md` constraints; `CONVENTIONS.md`; `apps/frontend/tailwind.config.js`
