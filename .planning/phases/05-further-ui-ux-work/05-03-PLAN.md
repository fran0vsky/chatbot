<objective>
Add a persistent header bar above every code block in assistant messages: a detected-language
label on the left and a copy button on the right (D-07..D-10). The bar is always visible.

ngx-markdown renders `<pre><code class="language-X">…</code></pre>` into the DOM after parsing.
There is no Angular template for those nodes, so the header bar is injected by DOM
post-processing in a `(ready)` handler on the `<markdown>` component. The handler queries the
component host for `<pre>` elements, and for each unprocessed one builds a header `<div>` and
inserts it as the first child of the `<pre>` (or as a wrapper sibling — executor's discretion;
inserting inside the `<pre>` before the `<code>` is simplest and keeps existing `.markdown-bubble pre`
styling). Each processed `<pre>` is marked with a data attribute so re-renders don't double-inject.

Language is read from the `<code>` element's `language-*` class. Copy uses `code.textContent`.
Copy feedback (checkmark for ~2s) follows the established Phase 3 pattern from `copyMessage()`.

Output: updated MessageBubble (ts/html/scss).
</objective>

  <files>libs/ui/src/lib/message-bubble/message-bubble.ts</files>

  <action>
    Add an `ElementRef` injection and an `onMarkdownReady()` method that builds a header bar for
    every code block. This is DOM post-processing — ngx-markdown's output is plain DOM with no
    Angular bindings, so building nodes imperatively is the correct approach here.

    1. Inject ElementRef alongside the existing ChangeDetectorRef:

         private readonly host = inject<ElementRef<HTMLElement>>(ElementRef);

       Add `ElementRef` and `inject` to the `@angular/core` import (inject is already imported).

    2. Add `onMarkdownReady(): void`:
       - `const pres = this.host.nativeElement.querySelectorAll('pre');`
       - For each `pre` (cast to HTMLElement):
         - Skip if `pre.dataset['headerInjected'] === 'true'` (idempotency guard).
         - Set `pre.dataset['headerInjected'] = 'true'`.
         - Find the code element: `const code = pre.querySelector('code');` — if none, skip.
         - Detect language: scan `code.classList` for a class starting with `language-`; the
           language is the substring after `language-`. Treat `language-none`, `language-text`,
           and an empty result as "no language". Lowercase it.
         - Build the header `<div>` with `document.createElement('div')`:
             header.className =
               'flex items-center justify-between px-3 py-2 ' +
               'bg-desert-parchment dark:bg-desert-night-parchment ' +
               'border-b border-desert-border dark:border-desert-night-border';
         - Build the language label `<span>`:
             label.className = 'text-xs font-mono text-desert-brown-muted dark:text-desert-night-muted';
             label.textContent = language ?? '';
           Append the label even when empty — it holds the left slot so the copy button stays
           right-aligned via `justify-between`.
         - Build the copy `<button>` (type="button"):
             button.className =
               'p-1 rounded text-desert-brown-muted dark:text-desert-night-muted ' +
               'hover:bg-desert-border dark:hover:bg-desert-night-border ' +
               'hover:text-desert-brown dark:hover:text-desert-night-text transition-colors';
             button.setAttribute('aria-label', 'Copy code');
           Set the button's innerHTML to the COPY icon SVG (the two-path copy icon used in the
           message-bubble copy button — copy it verbatim from message-bubble.html, `w-4 h-4`).
         - Wire the click handler — see Task 3 for copy behavior; for this task, call a private
           helper `this.copyCodeBlock(button, code)`.
         - `header.appendChild(label); header.appendChild(button);`
         - Insert the header as the first child of the `<pre>`:
             `pre.insertBefore(header, pre.firstChild);`

    3. Constraints:
       - No `any` — type `pres` via `querySelectorAll('pre')` (NodeListOf<HTMLPreElement>),
         use `HTMLElement`/`HTMLButtonElement`/`HTMLSpanElement` for created nodes.
       - Tailwind tokens only in `className` strings — no inline `style`. The desert tokens used
         here (`desert-parchment`, `desert-night-parchment`, `desert-border`,
         `desert-night-border`, `desert-brown-muted`, `desert-night-muted`, `desert-brown`,
         `desert-night-text`) all already exist in tailwind.config.js. IMPORTANT: because these
         classes only ever appear in a `.ts` string (not a template), Tailwind's content scanner
         must see them. Verify after build that the classes are not purged — if they are, add
         this file's path is already covered by the libs/ui tailwind `content` glob (it scans
         `**/*.{html,ts}`), so string literals are picked up; confirm in the build verification.
       - This runs after every markdown render; the `data-header-injected` guard makes repeat
         calls cheap and idempotent.
  </action>
