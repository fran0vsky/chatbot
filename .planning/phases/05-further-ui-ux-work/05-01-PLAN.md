<objective>
Add slide+fade entry animations to chat message bubbles. New bubbles that mount during an active
session animate in from their own side (user → from right, assistant/error/typing → from left) over
150ms. The greeting (initial render and after new-chat reset) appears instantly.

Approach: a pure-CSS keyframe animation gated by an `animate` input on `MessageBubble`. Because the
`@for` loop in chat.html uses `track $index`, each bubble's DOM element is created exactly once when
its message is appended — a CSS animation on that element therefore runs once at mount and never
replays. No JS animation state, no Angular animations import, no `markForCheck()` needed: the
`animate` input is bound at creation time and never changes for a given element.

The greeting is always `messages[0]`, so `[animate]="$index >= 1"` cleanly excludes it on both
initial load and after `newChat()` (which resets `messages` to a single greeting).

Output: updated MessageBubble (ts/html/scss) and chat.html.
</objective>
