<objective>
Let a user teach a dino a durable skill once, in a dedicated training flow, so it applies in every future chat with that dino — plus a management surface to review/delete what a dino has learned. Builds directly on the Phase 21 memory store.

Purpose: Persistent, user-authored capability per dino — "teach once, never re-teach".
Output: A `dino_skills` table, MemoryService skill methods, a SkillsController, the teach-a-skill UI flow, and a SkillManager component for MEM-06.
</objective>

<design_decisions>
- A "skill" = { title, instruction } authored via a guided chat. The simplest robust v1: the teach flow is a focused chat where the dino helps the user articulate the skill, and on "Save skill" the final instruction text is persisted. (Avoid over-engineering: a skill is durable prompt text, not executable code.)
- Skills inject as a labeled block ("Skills this user has taught you:") separate from auto-extracted memories, so the dino treats them as standing instructions.
- Management (MEM-06) lists both skills and memories with delete buttons; reuse MemoryService for memory list/delete from Phase 21.
- SkillManager is presentational (inputs + outputs, story); SkillService (smart) does HTTP.
</design_decisions>
