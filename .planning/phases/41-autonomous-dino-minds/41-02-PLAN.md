(topic analysis + governor speaker scheduling + intent constraints). In its place: each round, every
participant dino independently makes ONE decision call **on its own model** with its own persona and the
full thread so far, choosing answer / react / silent and (for answers) its own stance. Answers stream on
the dino's own model and reference what other dinos just said; reactions pin an emoji; silences cost
nothing. Enforce the Plan 01 cost ceiling and keep the existing SSE contract so the frontend is unchanged.

<design_decisions>
- **decideAction (D-01):** New private `async decideAction(profile, dino, state, roster, hasPriorDinoThisRound,
  signal): Promise<DinoDecision>`. Build a per-dino `ChatOpenAI({ model: dino.model, apiKey:
  process.env['OPENROUTER_API_KEY'], configuration: { baseURL: 'https://openrouter.ai/api/v1' } })`,
  invoke with `buildDecisionPrompt(...)` + the attributed thread, `parseDecision` the output. Whole body
  in try/catch returning `heuristicDecision(profile, hasPriorDinoThisRound)` on any error/abort. Mirrors
  the old `decideIntent` try/catch+signal shape (L262-317) but on the OWN model and with no allowed-set.
- **Round loop (D-02):** Replace the governor while-loop with:
  ```
  for roundIndex in 0..MAX_ROUNDS-1:
    answersThisRound = 0
    for dino in roster (selection order; @mentioned dinos first ONLY in round 0):
      if atTotalAnswerCap(counters): break out of all rounds
      hasPrior = any dino message exists this round
      decision = imageGenDino ? imageGenDecision(...) : await decideAction(...)
      if dinoAtAnswerCap(counters, dino.id) and decision.action==='answer': downgrade to 'react' or 'silent'
      apply decision: answer → stream+emit+push transcript+counters++; react → emit reaction; silent → nothing
      if decision.action==='answer': answersThisRound++
      if signal.aborted: return
    if shouldStopRounds(roundIndex, answersThisRound): break
  emit group_done
  ```
  Each completed answer is pushed onto `state.transcript` BEFORE the next dino decides, so sequential
  thread context (GRP3-02) is automatic.
- **@mentions (D-03):** Keep `parseMentions`. In round 0 only, mentioned dinos are moved to the front of
  the iteration order and forced to `action: 'answer'` (skip their decision call) — preserves the
  "@dino forces a reply" behavior. Later rounds use pure autonomous decisions.
- **Image-gen decision (D-04):** `imageGenDecision()` returns react (with `ARTIST_DEFAULT_REACTION`,
  targeting the most recent message) with some probability else silent — no LLM call. Keep the existing
  image-gen react emission shape (L412-420).
- **buildDirective adapted (D-05):** Keep `buildDirective` but change its first arg from a bare
  `SpeechIntent` to read `decision.intent ?? 'answer_user'`; drop the `TopicAnalysis` arg (no central
  topic anymore) — replace the `topicNote` with nothing or a thread-derived hint. Answer generation still
  passes the directive as the 10th `streamAgent` arg.
- **Reactions (D-06):** A `react` decision emits `{ type: 'reaction', dinoId, emoji: decision.emoji,
  targetMessageId: decision.replyToMessageId ?? <last message id> }`. The Phase 37 random "bystander
  reaction" (L495-504) is REMOVED — reactions are now first-class autonomous decisions, not random noise.
- **Empty plan kept (D-07):** Still `yield { type: 'plan', plan: { round1: [], round2: [] } }` first for
  contract stability; the frontend creates slots dynamically on `dino_token`.
- **Cost ceiling source (D-08):** Use only the Plan 01 constants; delete `defaultBudget`/`TurnBudget`.
  Track `RoundCounters` (per-dino answer counts + total). Worst case documented in CONTEXT.
- **Out of scope:** frontend changes (Plan 03), doc/cost-cap write-up (Plan 03), schema/DB.
</design_decisions>
