<design_decisions>
- **DinoDecision shape (D-01):** Put it in `group.types.ts` (next to `DinoTurnDecision`). `action` is the
  primary output; `intent` is the dino's own stance **only when action='answer'** (reuse `SpeechIntent`,
  default `answer_user`); `emoji` is required only for `react`; `replyToMessageId`/`replyToAgentId` point at
  the prior message the dino is responding to (enables the threaded reply stub); `confidence` 0..1.
- **Cost ceiling as flat constants (D-02):** `MAX_GROUP_DINOS = 4`, `MAX_ROUNDS = 3`,
  `MAX_ANSWERS_PER_DINO = 2`, `MAX_TOTAL_ANSWERS = 8`. These replace the governor's `TurnBudget`/
  `defaultBudget`. Export them so Plan 02 and the docs reference one source of truth.
- **buildDecisionPrompt (D-03):** Pure function returning `{ system: string; human: string }` (or a single
  combined string — implementer's choice, but keep it a pure return, no SystemMessage objects so it stays
  testable). System text establishes the dino fully in persona (name, personality, speakingStyle,
  expertise/weak areas, confidence) and the rules: choose EXACTLY ONE of answer/react/silent; answer only
  with something to add; react with a single emoji to a specific message; silent if you'd only echo; if
  answering, also pick your own stance from the SpeechIntent set and the message you're replying to.
  Demand ONLY a JSON object. Human text carries the attributed thread (already formatted upstream) +
  whether any dino has spoken this round.
- **parseDecision (D-04):** Strip ``` fences (reuse the L174-180 pattern), `JSON.parse`, then validate:
  `action ∈ {answer,react,silent}` else `silent`; if `answer`, `intent` ∈ SpeechIntent else `answer_user`;
  if `react`, require a non-empty `emoji` string else downgrade to `silent`; clamp `confidence` to 0..1.
  Return a fully-typed `DinoDecision`; on any parse failure return `{ action: 'silent', confidence: 0 }`.
- **heuristicDecision (D-05):** Deterministic fallback from `profile.interactionBiases` (model of
  `heuristicIntent` L320-344): high `talkativeness`/`likesToChallenge` → answer (with a stance derived
  from biases); low → react or silent. Always a valid `DinoDecision`. Used by Plan 02 when the own-model
  decision call throws/aborts.
- **Round/cost predicates (D-06):** Small pure helpers + a `RoundCounters` record (per-dino answer counts,
  total answers) with: `dinoAtAnswerCap(counters, dinoId)`, `atTotalAnswerCap(counters)`,
  `shouldStopRounds(roundIndex, answersThisRound)` → true when `roundIndex+1 >= MAX_ROUNDS` OR
  `answersThisRound === 0`. Keep them independently unit-testable.
- **Out of scope:** any LLM/network call (Plan 02), any change to `streamGroup`, deleting governor.ts
  (Plan 02), UI.
</design_decisions>
