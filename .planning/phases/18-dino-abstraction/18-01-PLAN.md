<objective>
Introduce the "dino" as a first-class backend concept: a registry entry bundling a fixed model + system prompt (personality, response style, workflow) + an allowed tool subset. Wire the agent loop to resolve a `dinoId`, inject the dino's system prompt, and enforce its toolset server-side. Expose the roster to the frontend via `GET /api/dinos` and a shared typed contract. Ship at least 4 distinct dinos. Keep existing model-only chat working so the current frontend doesn't break before Phase 19.

Purpose: This is the keystone of v2.0 — everything (picker, Explore, groupchat, arena, vision dino) is "a configured dino."
Output: Shared dino types, a backend dino registry, a dinos controller, and an agent loop that is dino-aware.
</objective>

@.planning/REQUIREMENTS.md
Current backend agent loop is a MANUAL loop (no LangGraph) in agents.service.ts. It is stateless per request: it builds `conversation = [new HumanMessage(message)]` and never injects a system message. Tools come from a flat array in tools/index.ts and are filtered by the client-supplied `enabledTools`. There is no conversation history sent today (a known gap, addressed in Phase 21). Postgres exists but is unused here.
</context>

<design_decisions>
- A dino's `systemPrompt` and `toolNames` live ONLY on the backend registry (DINO-01 single source of truth). The frontend never sends a system prompt.
- The frontend sends `dinoId`; the backend resolves model + system prompt + tools. This keeps personality/tooling tamper-proof from the client (DINO-04).
- `enabledTools` (client) is now an OPTIONAL further-narrowing filter, intersected with the dino's toolNames — it can never widen beyond the dino's allowed set.
- Backward compatibility: if no `dinoId` is sent, behave exactly as today (use `model` + `enabledTools`, no system prompt). This lets the current frontend keep working until Phase 19 switches it to dinos.
- Start with 4 text-only dinos using already-verified free models from chat.ts (`openai/gpt-oss-120b:free`, `openai/gpt-oss-20b:free`, `z-ai/glm-4.5-air:free`, `nvidia/nemotron-3-nano-omni-30b-a3b-reasoning:free`). The vision/artist dinos arrive in Phases 25–26.
</design_decisions>
