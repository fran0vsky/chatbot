| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon internal gaps, inline badge padding |
| sm | 8px | Button padding in action rows (p-1.5 = 6px — existing exception noted below) |
| md | 16px | Default element spacing, composer left-padding |

| 2xl | 48px | Major section breaks |

- Icon action buttons in message-bubble and input-composer use `p-1.5` (6px) — pre-existing project pattern for compact hover-reveal action rows; do NOT change to 8px.

## Component Inventory

### Modified Components

#### `InputComposer` (`libs/ui/src/lib/input-composer/`)

The voice placeholder button at position 3 in the left button group (`aria-label="Voice input"`, title="Voice (coming soon)") becomes functional.

**Mic button states:**

| State | Visual |
|-------|--------|
| `supported() = false` | Button hidden entirely (`@if (voiceService.supported())`) — do not show broken mic on Firefox |
| Idle (supported, not listening) | Icon: microphone SVG, color: `jungle-ink-muted dark:jungle-night-muted`, same `w-8 h-8` as sibling buttons |
| Listening (active) | Icon: animated mic with pulse ring — `animate-ping` on a `jungle-accent` ring; icon color changes to `jungle-accent dark:jungle-night-accent`; `aria-label="Stop dictation"` |
| Disabled (composer disabled) | `opacity-50 cursor-not-allowed` — matches existing textarea disabled pattern |

**New `@Output()`:** `dictate: EventEmitter<string>` — emits the final transcript on `isFinal`. Parent `ChatComponent` fills `InputComposer.draft` via `@ViewChild`.

The mic button does NOT auto-submit on final result — it fills the draft and lets the user review and send manually. (Prevents accidental sends from mishearing.)

#### `MessageBubble` (`libs/ui/src/lib/message-bubble/`)

Add a "read aloud" speaker icon button to the existing action row (the hover-reveal `opacity-0 group-hover:opacity-100` row). Positioned between the copy button and the share button (left of share).

**Read-aloud button states:**

| State | Visual |
|-------|--------|
| Idle | Icon: speaker SVG (`w-3.5 h-3.5`), color: `jungle-ink-muted dark:jungle-night-muted`, same `p-1.5 rounded-md` as sibling action buttons |
| Currently speaking this bubble | Icon: speaker with wave SVG (or speaker + animated dot), color: `jungle-accent dark:jungle-night-accent` |
| Another bubble is being spoken | Idle appearance — global speaking state is shown in header, not per-bubble |
| `ttsService.supported() = false` | Button hidden entirely |

**New `@Output()`:** `readAloud: EventEmitter<string>` — emits `message.text`. Parent wires to `VoiceSynthesisService.speak()`.

Only appears on `role === 'assistant'` messages (not user, not error, not tool-call bubbles).

### New Components (Angular Services — not UI components)

These are services, not visual components, but their observable state drives UI changes:

| Service | File | Signals exposed | UI consumer |
|---------|------|----------------|-------------|
| `VoiceSynthesisService` | `apps/frontend/src/app/voice/voice-synthesis.service.ts` | `speaking: Signal<boolean>`, `supported: Signal<boolean>` | `MessageBubble` (read-aloud button), `HeaderBar` (global speaking indicator) |
| `VoiceRecognitionService` | `apps/frontend/src/app/voice/voice-recognition.service.ts` | `listening: Signal<boolean>`, `transcript: Signal<string>`, `supported: Signal<boolean>` | `InputComposer` (mic button state, draft fill) |

### Global Speaking Indicator

When `VoiceSynthesisService.speaking()` is true, the `HeaderBar` (or fixed top bar) shows a minimal "Dino is speaking" indicator + a "Stop" button. This satisfies the RESEARCH.md recommendation ("global state via `VoiceSynthesisService.speaking` signal").

**Indicator placement:** Right side of header, adjacent to existing theme toggle. Appears/disappears via `@if (voiceService.speaking())`.

**Stop button:** `aria-label="Stop reading aloud"`, icon: square stop SVG (same stop icon used in send-button stop state: `rect x="4" y="4" width="12" height="12"`), color: `jungle-accent dark:jungle-night-accent`.

## Interaction Contract

### Mic Button — Dictation Flow

1. User taps mic button → `VoiceRecognitionService.start()` called
2. Button transitions to listening state (pulse ring, accent color)
3. Interim transcript populates `InputComposer.draft` live as user speaks
4. On `isFinal`: `listening` signal drops to false, button returns to idle
5. User reviews draft and presses Send (or edits)
6. If user taps mic again while listening: calls `stop()` → `onend` fires → listening drops

Pressing mic while TTS is playing does NOT stop the TTS first. Both can run simultaneously (unlikely in practice but not prevented — browser handles mic capture independently of speaker output).

### Read-Aloud Flow

1. User hovers assistant message → action row becomes visible
2. User clicks speaker button → `readAloud.emit(message.text)` → parent calls `VoiceSynthesisService.speak(text)`
3. Global header indicator appears ("Dino is speaking..." + Stop button)
4. Speaker button on the active bubble shows speaking state (accent color)
5. On utterance end → `speaking` signal drops → header indicator disappears → bubble button returns to idle
6. Clicking Stop (header) → `VoiceSynthesisService.stop()` called → `speaking` drops immediately

### Accessibility

- Mic button: `aria-label` switches between "Start dictation" (idle) and "Stop dictation" (listening). `aria-pressed` bound to `listening` signal.
- Read-aloud button: `aria-label` switches between "Read aloud" (idle) and "Stop reading" (speaking this bubble). `aria-pressed` bound to `voiceService.speaking() && thisIsActive`.
- `SpeechRecognition` permission: browser prompts natively; no custom permission UI needed.
- Mic button hidden on Firefox: no broken UI state for unsupported browsers.
- Privacy tooltip: small `title` attribute on mic button: "Voice input — audio is processed by your browser (Google on Chrome, Apple on Safari)". No modal required for a portfolio app.
