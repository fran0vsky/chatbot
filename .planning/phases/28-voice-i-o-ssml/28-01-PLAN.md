<objective>
Deliver VOX-01 (read responses aloud) and VOX-02 (SSML-driven prosody, honestly handled) as a complete vertical slice. A user hovers an assistant message, clicks a speaker button, and hears the dino read it aloud in that dino's voice character; a global header indicator shows "Dino is speaking…" with a Stop button. Prosody comes from an `SsmlHint` struct mapped to `SpeechSynthesisUtterance` properties — never raw SSML XML, which the browser speaks literally (RESEARCH.md Pitfall 1). A `TtsProvider` interface is the clean seam for a future paid real-SSML adapter.

Purpose: Two-way voice — the dino can talk back — using the free, browser-native Web Speech API, no backend round-trip.
Output: A `providedIn:'root'` VoiceSynthesisService + TtsProvider/BrowserTtsAdapter, a per-dino `voiceProfile` in the registry, a read-aloud button on MessageBubble, and a global speaking indicator in the chat header.
</objective>

<design_decisions>
- VOX-02 honest handling (D-05): SpeechSynthesis does NOT process SSML. Define `SsmlHint` (a plain TS struct: rate/pitch/volume/pauseAfterMs) and map it to utterance properties. A code comment in browser-tts-adapter.ts documents the limitation citing browser-compat-data #15663. Never pass `<speak>`/`<prosody>` to `utterance.text`.
- `voiceProfile` lives in the backend dino registry (D-02), typed in `@org/shared-types`. Shape: `rate` (0.1–10, default 1.0), `pitch` (0–2, default 1.0), `preferredVoice?` (system voice name/URI). Missing profile → defaults; `preferredVoice` not installed → system default voice (D-03, never error).
- Manual read-aloud only this phase (D-04): user clicks the speaker button. No auto-read.
- Global speaking state via `VoiceSynthesisService.speaking` signal drives the header indicator + Stop (D-10); the active bubble shows a local speaking state.
- Hand-roll the thin service (RESEARCH.md recommendation) rather than add `@ng-web-apis/speech`. `providedIn:'root'` so Phase 29 reuses it.
- Always `synth.cancel()` before `synth.speak()` (Pitfall 6). Load voices lazily at `speak()` time, not at construction (Pitfall 2).
- Seed Phase 29: wire the existing `read_last_message` catalogue intent to actually speak via the service, so the voice assistant can later drive read-aloud through the same surface.
</design_decisions>
