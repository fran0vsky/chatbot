<objective>
Deliver VOX-03 (dictate input by voice) as a complete vertical slice. The placeholder "Voice input" button in the composer becomes a functional microphone: the user taps it, grants the browser mic permission, speaks, and watches their words populate the composer draft live; on the final result the mic returns to idle and the user reviews and sends. The button is hidden on browsers without SpeechRecognition (Firefox) so there is never a broken state.

Purpose: The other half of two-way voice — talk to the dino instead of typing — using the free, browser-native SpeechRecognition API.
Output: A `providedIn:'root'` VoiceRecognitionService (NgZone-wrapped, signal-based) and a functional mic button on InputComposer wired into ChatComponent's draft.
</objective>

<design_decisions>
- Hand-roll VoiceRecognitionService per RESEARCH.md Pattern 2: `interimResults = true`, `lang = 'en-US'`, `continuous = false` (single-utterance dictation). Feature-detect `SpeechRecognition || webkitSpeechRecognition`; `supported.set(false)` and hide the mic when absent (Pitfall 3, D-09).
- Guard `start()` with the `listening` signal to avoid `InvalidStateError` (Pitfall 5).
- Wrap onresult/onend/onerror in `NgZone.run()` so OnPush components re-render (RESEARCH.md key constraint).
- STT fills draft, never auto-submits (D-08). Treat the transcript as untrusted: trim + enforce the same max-length as typed input before it reaches onSend() (RESEARCH.md Security Domain, V5).
- Privacy: small `title` tooltip on the mic button noting audio is processed by the browser's cloud ASR (Pitfall 4). HTTPS required in prod; localhost works in dev (Pitfall 7) — document in a code comment, no runtime action.
- `providedIn:'root'` so Phase 29's voice assistant reuses the same recognition service.
</design_decisions>
