<objective>
Build the backend AI engine for the Memory Creator: three server-side operations that (1) **suggest**
â‰Ą3 things worth remembering from the current conversation, (2) **synthesize** a chosen suggestion or
free natural text into the 3-field skill shape (name / when-to-activate / instruction), and (3)
**reconcile-and-save** the item â€” comparing it to the user's existing skills and either creating a new
skill or updating an overlapping one, with the new-vs-update decision made entirely server-side.

All LLM calls run on the **active dino's own model** (resolved server-side via `getDino(dinoId)`) and
reuse the existing **paid-fallback** pattern so free-model 429s never break the creator. Image-gen dinos
fall back to a text model. The creator targets **DinoSkills only** (D-01) and leaves the background
`userMemories` extraction pipeline (`extractAndStoreMemories`) completely untouched (D-02).

Purpose: give the Phase 34 frontend creator (plan 02) a real API to generate suggestions, fill the form,
and persist with auto-reconciliation.
Output: shared creator contracts in `dino.types.ts`, a new `MemoryCreatorService` + `MemoryCreatorController`
wired into `MemoryModule`, and unit tests for the parsing + reconcile-routing logic.
</objective>

<design_decisions>
- **Routes (D-08):** add a sibling controller `MemoryCreatorController` so `SkillsController` stays focused.
  Routes under the global `api` prefix: `POST skills/suggest`, `POST skills/synthesize`, `POST skills/save`.
  These do not collide with the existing `@Post('skills')` / `@Delete('skills/:id')`. No new persistence
  endpoint â€” save reuses `addSkill`/`updateSkill` internally.
- **Model selection (D-04):** `buildModel(dino)` returns the dino's own `model`, EXCEPT when `dino.imageGen`
  is true (image-output models can't do the text task) â€” then use `FALLBACK_MODEL`. Every LLM call goes
  through `invokeWithFallback(primaryModel, messages)`: try the primary, and on a capability/rate error
  (mirror `isCapabilityError`: 402/429/rate limit/no endpoints/quota/context length/model not found) retry
  once on `FALLBACK_MODEL`. Construct `ChatOpenAI` exactly like `extractAndStoreMemories` does
  (apiKey from `process.env['OPENROUTER_API_KEY']`, `baseURL: https://openrouter.ai/api/v1`).
- **Suggest (SC#1):** system instruction asks for â‰Ą3 short, distinct "things worth remembering about how
  this user wants you to behave" derived from the conversation, one per line. Parse with the same line
  cleanup `extractAndStoreMemories` uses (`replace(/^[-*\d.)\s]+/, '')`, drop blanks). Return `string[]`.
- **Synthesize (SC#2):** one LLM call turns the chosen suggestion OR free text into the 3 fields. Prompt
  the model to reply as strict JSON `{ "title": string, "whenToActivate": string, "instruction": string }`
  (whenToActivate may be ""). Parse defensively (strip ``` fences, `JSON.parse`, fall back to a minimal
  object using the raw input as instruction if parsing fails). Return `SynthesizedSkill`.
- **Reconcile + save (D-07/D-08):** `reconcileAndSave(userId, dinoId, item)` first calls
  `memory.getSkills(userId, dinoId)`. If there are existing skills, one LLM reconcile call is given the new
  item + the existing skills (id + title + instruction) and must reply strict JSON
  `{ "decision": "new" | "<existingId>", "mergedTitle"?, "mergedWhenToActivate"?, "mergedInstruction"? }`.
  When `decision` is an id that matches an existing skill â†’ `updateSkill(id, { title: mergedâ€¦, â€¦ })` (folds
  the +/- delta into the merged instruction) â†’ action `"updated"`. Otherwise â†’ `addSkill(...)` â†’ `"created"`.
  No existing skills â‡’ skip the reconcile call, always create. A `null` from add/update (DB off) â†’
  `ServiceUnavailableException`. The decision is never returned to the client as a toggle â€” only the final
  skill + a plain `action` string.
- **Validation:** controller rejects missing `userId`/`dinoId` (and `input` for synthesize, `title`/
  `instruction` for save) with `BadRequestException`. Unknown `dinoId` (`getDino` returns undefined) â†’
  `BadRequestException('Unknown dinoId')`.
- **Graceful LLM failure:** if an LLM call throws even after fallback, `suggest` returns `[]` and the
  controller responds normally (the UI shows an empty/thinking-failed state); `synthesize` falls back to
  `{ title: '', whenToActivate: '', instruction: input }`. A creator failure must never 500 the chat.
</design_decisions>
