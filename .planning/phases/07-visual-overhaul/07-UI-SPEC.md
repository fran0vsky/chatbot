# Phase 7 — UI Design Contract

> Visual and interaction contract for Phase 7: Visual Overhaul.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase is a complete visual overhaul of every visible UI component. The
> western/desert theme (sand/amber/brown palette, cactus silhouettes, day/night
> toggle) is PRESERVED and ELEVATED — never replaced. The benchmark for polish
> is ChatGPT, Claude, and Gemini. No backend, no new features, no routing changes.
>
> Surface areas covered:
> 1. Layout shell — `chat.html`, `<main>` stacking, scroll regions
> 2. History sidebar — `HistoryPanel` (overlay drawer)
> 3. Header bar — `HeaderBar`, `ThemeToggle`, `NewButton`, `ModelSelector`
> 4. Message list — `MessageBubble` (user / assistant / error / typing)
> 5. Input — `InputComposer` (floating pill)
> 6. Typography scale, spacing system, scrollbar, empty/loading states, micro-animations

## Spacing Scale

Declared values (all multiples of 4 — Tailwind 8-point scale):

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `1` / `gap-1` | Icon-to-label gaps, copy button inner padding |
| sm | 8px | `2` / `gap-2` / `py-2` | Compact element spacing, pill inner row gap, header control gaps |
| md | 16px | `4` / `px-4` / `py-4` | Default — chat area horizontal padding, header padding, pill horizontal padding, message vertical rhythm |
| lg | 24px | `6` / `gap-6` / `py-6` | Section breaks — empty-state vertical gap, sidebar section padding |
| xl | 32px | `8` | Layout gaps, large vertical breathing room |
| 2xl | 48px | `12` / `pb-12` | Empty-state bottom offset (landing view) |
| 3xl | 64px | `16` | Reserved — page-level spacing if needed |

**Exceptions (intentional, documented):**
- Message column width: `max-w-2xl` (672px) centered — caps line length for readability, matches ChatGPT/Claude. Applies to message list AND input pill (consistent column).
- Message bubble width: `max-w-[75%]` within the column — preserves clear left/right asymmetry.
- Sidebar drawer width: `w-72` (288px) — fixed overlay drawer width.
- Send button touch target: `w-9 h-9` (36px) circular — between the 32px chrome buttons and a 44px ideal; keep as-is, it reads as the primary action.
- Header/sidebar icon buttons: `w-8 h-8` (32px) — established chrome control size; keep consistent across `NewButton`, `ThemeToggle`, history toggle.
- Inter-message vertical rhythm: `space-y-5` (20px) between bubbles in chat state. 20px is a deliberate non-8 step for comfortable message density (Claude-like spaciousness without sprawl). This is the ONLY non-8 spacing value and is locked here.
- Sticky input bottom offset: `sticky bottom-4` (16px gap above viewport edge) with `pb-4` — floating pill never touches the screen edge.

Source: 05-UI-SPEC.md + 06-UI-SPEC.md spacing baselines; current `chat.html` (`max-w-2xl`, `space-y-5`, `sticky bottom-4`); current `input-composer.html` (`px-4 py-3`, `w-9 h-9`).

## Typography

Exactly 4 sizes, exactly 2 weights. No new sizes or weights introduced.

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Display / heading | 18px (`text-lg`) | 600 semibold | 1.2 | Cinzel `font-title` | Header `CHATBOT` title (uppercase); empty-state greeting heading |
| Body / chat | 14px (`text-sm`) | 400 regular | 1.625 (`leading-relaxed`) | Inter `font-body` | Assistant + user message text, textarea input, sidebar session titles, model selector |
| Subheading / label | 14px (`text-sm`) | 600 semibold | 1.4 | mixed | Sidebar `History` header (`font-title` + uppercase + `tracking-wide`); "New chat" button label |
| Caption / meta | 12px (`text-xs`) | 400 regular | 1.4 | Inter `font-body` | Session timestamps, code-block language badge, sr-only text, empty-state subtext |

**Rules:**
- Two weights only: regular (400) + semibold (600). No `font-medium` (500), no `font-bold` (700). The sidebar "New chat" button currently uses `font-medium` — **change to `font-semibold`** to comply.
- Body line-height standardised at `leading-relaxed` (~1.625) for all chat text — matches Claude's airy message blocks. Sidebar list uses `leading-snug` (~1.375) for compact density — acceptable exception for the chrome list.
- Title elements keep `uppercase`. Cinzel at 600 reads monumental — header stays `text-lg`.
- No size between 14 and 18 — the 16px slot is intentionally unused to keep the scale tight.

Source: current `header-bar.html` (`text-lg font-semibold uppercase font-title`); `message-bubble.html` (`text-sm font-body leading-relaxed`); `history-panel.html`; 06-UI-SPEC.md typography baseline.

### Layout contract — Chat shell

- `<main>` is `relative flex flex-col`; cactus silhouettes are `absolute … z-0`; all message + input content sits at `z-10`. **Unchanged — preserve.**
- **Landing state** (`messages.length <= 1 && !isLoading`): vertically centered greeting bubble + input composer, both in a `max-w-2xl mx-auto` column, `gap-6`, `pb-12` bottom offset.
- **Chat state**: scrollable message list (`chat-scroll-area`, `max-w-2xl mx-auto`, `space-y-5`) above a sticky floating input pill (`sticky bottom-4`, gradient fade `from-desert-sand`/`from-desert-night to-transparent`).
- The gradient fade behind the sticky pill must remain so messages dissolve under the pill rather than colliding with it.

### Sidebar contract — HistoryPanel

- Overlay drawer: `fixed inset-y-0 left-0 w-72 z-40`, slides via `translate-x-0` / `-translate-x-full`, `transition-transform duration-300 ease-in-out`. **Preserve.**
- Backdrop scrim: `fixed inset-0 bg-black/40 z-30`. **Preserve.**
- Session row: `flex items-start gap-2 px-3 py-2 mx-1 my-0.5 rounded` — keep, refine hover.
- Active row: `bg-desert-parchment dark:bg-desert-night-parchment` + `aria-current="true"`.
- Inactive hover: `hover:bg-desert-border dark:hover:bg-desert-night-border`.
- Delete button: `opacity-0 group-hover:opacity-100` — reveal on row hover only.

### Message bubble contract

| Variant | Alignment | Background | Radius | Padding |
|---------|-----------|------------|--------|---------|

| Error | left | `red-50` + `red-200` border | `rounded-2xl rounded-bl-sm` | `px-4 py-2` |
| Typing | left, with snake avatar | secondary parchment | `rounded-2xl rounded-bl-sm` | `px-4 py-2` |

- The single-corner-flattened radius (`rounded-br-sm` / `rounded-bl-sm`) is the speech-tail cue — preserve on every variant.
- Assistant bubbles carry the snake mascot avatar (`/assets/creatures/creature.png`, `w-16`, `image-rendering: pixelated`) — keep. The `style="image-rendering: pixelated"` inline attribute is the one permitted inline-style exception (no Tailwind equivalent); all other styling stays Tailwind utilities.
- Shadow: `shadow-md` day, `dark:shadow-black/30` night — keep for gentle elevation.

### Input pill contract

- Pill card: `w-full rounded-2xl shadow-xl bg-desert-sand-light dark:bg-desert-night-surface border border-desert-gold/50 dark:border-desert-night-border px-4 py-3`, inside the `max-w-2xl` column.

## Animation Contract

Micro-animations are in scope. All motion stays subtle, fast, and `ease-out` — never bouncy or attention-grabbing. Inherited tokens are kept; this phase may refine timing values but introduces no large or looping motion.

| Element | Animation | Duration | Easing | Status |
|---------|-----------|----------|--------|--------|
| New user bubble | Slide-in from right + fade | 150ms | `ease-out` | Inherited (`msg-enter-right`) — preserve |
| New assistant bubble | Slide-in from left + fade | 150ms | `ease-out` | Inherited (`msg-enter-left`) — preserve |
| Typing indicator bubble | Slide-in from left + fade | 150ms | `ease-out` | Inherited — preserve |
| Typing dots | 3-dot bounce, staggered 0/150/300ms | loop | default | Inherited (`animate-bounce` + `dot-delay-*`) — preserve |
| Sidebar drawer | Slide `translate-x` | 300ms | `ease-in-out` | Inherited — preserve |
| Backdrop scrim | Fade in/out with drawer | 300ms | `ease-in-out` | May add `transition-opacity` for smoother fade — refinement |
| Icon buttons (header, sidebar) | Background + color on hover | ~150ms | `transition-colors` | Inherited — preserve |
| Send button | `active:scale-95` press | instant | `transition` | Inherited — preserve |
| Send button hover | Background tone shift | ~150ms | `transition` | Inherited — preserve |
| Session row hover | Background tone shift | ~150ms | `transition-colors` | Inherited — preserve |
| Cactus silhouettes | Static — no animation | — | — | Decorative only — never animate |
| Snake mascot | Static — no animation | — | — | Decorative only |

**Rules:**
- No new keyframe animations beyond the existing slide-fade and dot-bounce.
- Respect reduced-motion: entry animations should be wrapped so `prefers-reduced-motion: reduce` disables transform-based motion (executor adds the media-query guard to the existing `msg-enter-*` keyframes in `message-bubble.scss`).
- OnPush constraint: any flag toggle that drives a class binding (e.g. `animate`) must call `cdr.markForCheck()` — established pattern in `message-bubble.ts`.

Source: `message-bubble.html` (`msg-enter-*`); `typing-indicator.html` (`dot-delay-*`); `history-panel.html` (`duration-300 ease-in-out`); 05/06-UI-SPEC.md animation contracts.

## Constraints Summary

All Phase 7 UI work must comply with:

1. **Theme preservation** — the desert/western palette, cactus silhouettes, snake mascot, and day/night toggle are PRESERVED and elevated. No palette replacement, no new color identity.
2. **Tailwind-only** — no inline styles, with one documented exception: `style="image-rendering: pixelated"` on the snake mascot `<img>` (no Tailwind utility exists). All other styling uses Tailwind utilities or `desert-*` / `cactus-*` tokens.
3. **No arbitrary hex in templates** — all colors via named tokens. Cactus/snake SVG `fill="..."` hardcoded hex is the established pixel-art exception.
4. **Angular standalone + OnPush** — no new NgModules. Any class-binding flag toggle must call `cdr.markForCheck()`.
5. **Presentational component naming** — `libs/ui` components carry no domain words, take no injected services. This phase adds no services and no domain-named components.
6. **No new `@Input`/`@Output`** — presentation-only overhaul; component public APIs are frozen.
7. **No backend / feature / routing changes** — `chat.service.ts`, `history.service.ts`, and `app.routes.ts` are untouched.
8. **`darkMode: ['class', '.night-mode']`** — all dark variants use the `dark:` prefix; no `prefers-color-scheme` for theming. `prefers-reduced-motion` IS used (motion-only guard).
9. **Typography discipline** — exactly 4 sizes (12/14/14/18px) and 2 weights (400/600). Replace any stray `font-medium` with `font-semibold`.
10. **60/30/10 color discipline** — accent (terracotta/amber) restricted to send button + user bubbles only.
11. **Accessibility** — visible `focus-visible` rings on all interactive controls; reduced-motion respected; existing `aria-*` and `sr-only` text preserved.
12. **TypeScript strict mode** — `noImplicitAny`, `strictTemplates`; no `any`, no `console.log`.
13. **No functional regressions** — chat send, model selector, theme toggle, new chat, history switching/deletion, copy buttons, code-block headers, auto-resize all unchanged.

Source: `apps/frontend/CLAUDE.md`; root `CLAUDE.md`; user MEMORY (Angular component architecture rule); 07-CONTEXT.md constraints.
