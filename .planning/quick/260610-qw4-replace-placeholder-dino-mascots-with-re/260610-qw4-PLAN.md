# Quick Task 260610-qw4

Replace the tinted-Spino placeholder dino mascots with real generated pixel-art
(staged in `dinos/_incoming/`) and wire per-dino head avatars into the chat UI to
match modern chatbots (avatar in header + beside each assistant message). Body
mascots stay in the Explore/picker gallery.

## Task 1 — Flood-fill keyer in split-mascot.js
- **Files:** scripts/split-mascot.js
- **Action:** Replace the per-pixel black-threshold + partial-alpha logic in
  `processHalf` with flood-fill background removal: BFS from the four image edges
  over near-black pixels (sum < ~70), set only those to transparent, leave all
  interior pixels fully opaque. Preserve the stacked-dual contract (day top half,
  night bottom half) and the `--all` / single-id / no-arg Spino entry points.
- **Verify:** Re-split rexford; composite onto magenta → body fully solid, no
  see-through (regression vs the partial-alpha bug).
- **Done:** `node scripts/split-mascot.js rexford` yields solid keyed bodies.

## Task 2 — Process all 6 dinos
- **Files:** apps/frontend/public/spino/dinos/_src/{id}-dual.png,
  dinos/{id}-day.png, dinos/{id}-night.png, dinos/avatars/{id}.png
- **Action:** For each of the 6 dinos: stack `_incoming/{id}-day.png` over
  `{id}-night.png` → `_src/{id}-dual.png`; run the flood-fill split → bodies sized
  ≤800px; flood-fill key `_incoming/{id}-head.png` → 256² transparent →
  `dinos/avatars/{id}.png`. Overwrite the placeholders.
- **Verify:** All 18 outputs exist; spot-check magenta + 48px for 2 dinos.
- **Done:** dinos/ has 6 real day+night pairs; dinos/avatars/ has 6 heads.

## Task 3 — Wire per-dino avatars
- **Files:** libs/ui/src/lib/mascot/mascot.ts + .html,
  libs/ui/src/lib/message-bubble/message-bubble.ts + .html,
  apps/frontend/src/app/chat/chat.ts + chat.html
- **Action:**
  - Mascot: add `@Input() avatar` mode that renders the single
    `/spino/dinos/avatars/{id}.png` (circle, pixelated), Spino-SVG fallback on error.
  - Chat header: switch the active-dino `<app-mascot>` to `avatar` mode.
  - message-bubble: add `@Input() avatarSrc` (default `/spino/spino-avatar.png`)
    used by the assistant-bubble `<img>`, with (error) fallback to the default.
  - chat.ts: add `activeDinoAvatarSrc` computed; chat.html passes it to every
    assistant `app-message-bubble` (incl. empty-state, typing, streaming bubbles).
- **Verify:** `nx build frontend` (or lint) passes; header + assistant bubbles
  show the active dino's head; picker/dino-card still show bodies.
- **Done:** Avatars render per-dino; body gallery unchanged.
