# Quick Task 260610-fp1: Force dino pick on load (no no-dino state)

## Problem

On a fresh page load or reload the app boots with `activeDinoId: undefined`
(`dino.reducer.ts`) and a welcome session, but the dino picker only opens via
`newChat()`. This leaves the user chatting against no bound dino — an invalid
state. The picker should greet the user on load (like the "New chat" flow) and
must not be dismissable until a dino is chosen.

## Tasks

1. **Force picker open on load** — `apps/frontend/src/app/chat/chat.ts`
   - Add a constructor `effect()` that opens the picker once the roster has
     loaded (`dinos().length > 0`) while `activeDinoId()` is undefined.
   - Reuses the existing `UiActions.openPicker()` path; no new state.

2. **Make the picker mandatory while no dino is bound** —
   `apps/frontend/src/app/chat/chat.ts` + `chat.html`
   - Add `dismissPicker()` — a no-op when `activeDinoId()` is falsy, otherwise
     delegates to `closePicker()`.
   - Backdrop click → `dismissPicker()` (was `closePicker()`).
   - Hide the close (X) button behind `@if (activeDinoId())`.
   - `pickDino()` still calls `closePicker()` directly, so selecting always closes.

## Verify

- `nx build frontend` succeeds.
- Fresh load with no dino → picker opens automatically; backdrop/X cannot
  dismiss it.
- After picking a dino, the picker closes; opening it again from "New chat"
  shows the X and is dismissable.
